# 排版标准

## 一、视觉层级

| 层级 | 样式 | 用途 |
|------|------|------|
| L1 | 【】 | 大模块标题 |
| L2 | ‖ | 子主题分隔 |
| L3 | **加粗** | 段落核心 |
| L4 | - | 列表要点 |

## 二、段落标准

**黄金法则：每段 ≤ 4 行**

```
段落结构：[核心句] + [支撑句] + [补充]
核心句：一句话讲明白
支撑句：2-3行解释
```

## 三、符号系统

```
【】大模块 → "【主题】内容"
‖ 分隔线 → 段落内小分隔
→ 流程步骤 → "→ 步骤1 → 步骤2"
- 列表要点
```

## 四、重点技巧

| 技巧 | 示例 |
|------|------|
| 首句原则 | 每段第一句是核心观点 |
| 数字具体化 | "87%的用户" > "很多用户" |
| 主动语态 | "我已修复" > "错误已修复" |
| 动词前置 | 把行动点放句首 |

## 五、长度控制

| 类型 | 最优 | 最大 |
|------|------|------|
| 简单请求 | 3-5行 | 10行 |
| 常规沟通 | 10-15行 | 30行 |
| 复杂说明 | 20-30行 | 50行 |
| 长篇报告 | 用附件 | 用附件 |

**3秒原则**：
- 3秒：看懂核心
- 10秒：了解全部
- 30秒：决定行动

## 六、结构模板

```
【主题】- 一句话说明白
【开头】- 1句话表明意图
【正文】- 核心内容（分块）
【结尾】- 行动点/期待
【署名】- 简洁
```

## 七、检查清单

**结构**
- [ ] 主题行清晰
- [ ] 开头一句说明意图
- [ ] 逻辑顺序正确
- [ ] 结尾有行动点

**段落**
- [ ] 每段 ≤ 4 行
- [ ] 段间有空行
- [ ] 每段首句是核心

**内容**
- [ ] 无术语/行话
- [ ] 数字已具体化
- [ ] 主动语态
- [ ] 动词前置

## 场景模板库

### 进度汇报

```
【主题】[进度] 项目名 - 第X周

‖ 当前状态
→ 已完成：功能A、功能B
→ 进行中：功能C（预计周五完成）
→ 风险点：无 / 有（简述）

‖ 下周计划
→ 任务1
→ 任务2

‖ 需要支持
→ 需要XX协助
```

### 决策请求

```
【主题】[请求] 确认：方案A vs 方案B

‖ 背景
一句话说明为什么需要决策

‖ 选项对比
|         | 方案A    | 方案B    |
| 优点    | xxx      | xxx      |
| 缺点    | xxx      | xxx      |

‖ 建议
倾向于X，因为...

‖ 请确认
截止时间：周五
```

### 问题升级

```
【主题】[问题] 简要描述 - 需要协助

‖ 问题
一句话定义问题

‖ 影响
- 影响范围：xxx
- 严重程度：高/中/低

‖ 已尝试
- 方案1：结果
- 方案2：结果

‖ 需要
- 需要XX协助
- 期望时间：今天
```
