# Book Processor - 书籍深度处理 Skill

> 📚 将 PDF 书籍转换为带批注、总结、中英对照的深度学习笔记

---

## 技能描述

本 Skill 提供一套完整的书籍处理流程，将 PDF 扫描版书籍转换为结构化的 Markdown 学习笔记，包含：
- 章节自动拆分
- 段前批注总结
- 核心概念提炼
- 表格整理
- 思考题设计

---

## 使用场景

- 📖 处理 PDF 扫描版书籍/课程课件
- 📝 创建结构化学习笔记
- 🎯 为 AI 对话准备知识库
- 📚 批量处理系列书籍

---

## 操作流程

### Step 1: PDF → Markdown

```bash
# 安装 poppler（如未安装）
brew install poppler

# 转换 PDF 为 Markdown
pdftotext -layout input.pdf output.md
```

### Step 2: 识别章节结构

让 AI 分析目录和章节标题，确定拆分点。

**Prompt 示例**：
```
请分析这份文档的章节结构，找出：
1. 共有几个章节
2. 每章的起始位置
3. 章节层级关系
```

### Step 3: 按章节拆分

```bash
# 按页码拆分
pdftotext -layout -f 起始页 -l 结束页 input.pdf chapter-XX.md
```

### Step 4: 单章处理（核心）

对每个章节，让 AI 进行深度处理：

**Prompt 模板**：
```
请对这份内容进行深度学习处理：

1. **段前批注**：每段内容前添加 💡📌🎯 等图标标记核心洞察
2. **表格整理**：将列表、对比内容转为表格
3. **图表可视化**：用 ASCII 图/mermaid 展示逻辑关系
4. **核心概念**：提炼关键术语和定义
5. **本章小结**：总结核心知识点
6. **思考题**：设计 3-5 道反思题

输出格式要求：
- 使用 Markdown
- 保持中英文对照（如适用）
- 批注简洁有力，不超过 50 字
- 表格清晰易读
```

### Step 5: 批量处理

让 AI 基于第一章的处理经验，批量处理剩余章节。

**Prompt 示例**：
```
基于第一章的处理风格和格式，请处理第二章、第三章...
保持：
- 相同的批注风格
- 相同的表格格式
- 相同的小结结构
```

### Step 6: 质量审查

检查处理结果：
- [ ] 批注是否准确精炼
- [ ] 表格是否清晰
- [ ] 逻辑是否连贯
- [ ] 思考题是否有启发性

---

## 输出格式模板

```markdown
# 第 X 章 章节标题

> 📝 **本章导读**：用一句话概括本章核心。

---

## 核心概念

> 💡 **定义**：关键概念的定义。

| 维度 | 说明 | 关键词 |
|------|------|--------|
| L1 | 描述 | 关键词 |

---

## 本章小结

### 核心知识点回顾

1. 知识点 1
2. 知识点 2

### 思考题

1. 问题 1
2. 问题 2
```

## 最佳实践

### ✅ 推荐做法

- 先处理一章作为样本，确认风格后再批量
- 保持批注简洁，避免喧宾夺主
- 表格优于大段文字
- 思考题要能引发深度反思

### ❌ 避免做法

- 批注过长，超过原文
- 过度解读，偏离原意
- 格式花哨，影响阅读
- 只处理不复习
