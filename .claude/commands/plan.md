# /plan - 启动五文件工作流

**作用**：一键初始化复杂项目的五文件工作流

---

## 用法

在项目中输入：

```
/plan
```

或

```
启动五文件工作流
```

---

## 执行流程

1. 在当前目录创建 `docs/` 文件夹
2. 复制五个模板文件到 `docs/`
3. 引导用户填写 `prompt.md` 和 `plans.md`
4. 准备就绪后，AI 按 `plans.md` 执行

---

## 文件结构

```
项目根目录/
└── docs/
    ├── prompt.md              # 项目定义（必填）
    ├── plans.md               # 任务拆解 + 验收（必填）
    ├── architecture.md        # 技术锁定（选填）
    ├── implement.md           # 执行手册（选填）
    └── documentation.md       # 进度日志（AI 自动更新）
```

---

## 核心机制

### 1. 防跑偏

- `prompt.md` 中的**排除项**明确不要什么
- `plans.md` 中的**验收命令**强制验证

### 2. 小步快跑

- 每个里程碑控制在 1-2 小时
- 验证不通过禁止进入下一阶段

### 3. 持久化记忆

- 会话间隔通过文件恢复上下文
- 错误记录到文件防止重复踩坑

---

## 何时使用

**适合**：
- 复杂开发项目（多模块、多阶段）
- 需要长期维护的项目
- 需要 25 小时 + 无人值守开发

**不适合**：
- 简单问答
- 单文件修改
- 快速查询
